# Kinolu UI 生成提示词手册（给 Gemini / Figma Make）

本文件用于你直接喂给 Gemini/Figma Make 生成原型图。  
目标不是写前端代码，而是先稳定视觉和交互母版。

---

## 0. 目标和边界

- 目标：基于你已认可的拍照页风格，扩展出其他页面原型。
- 输出：每次只生成 1 张图（移动端优先，390x844）。
- 边界：不生成代码、不发散重设计、不加入算法下拉和多模式选择。

---

## 1. 输入素材（每次生成前都要附带）

你每次都给模型 3 组参考：

1. **风格母版（必须）**  
   - Figma：`Camera Screen (Community)`  
   - 链接：`https://www.figma.com/design/0COl9mk72JhRXlV3LsZ7t2/Camera-Screen--Community-?node-id=3-38&t=XpYBVa0DxbS7zPQd-4`
   - 用途：锁定导航、按钮、边框、圆角、玻璃质感。

2. **交互参考（建议）**  
   - `ref/colorby/*`（结果页与调色流逻辑）
   - `ref/kumo/*`（拍照和入口动线）

3. **你当前想生成的页面目标**  
   - 例如：Home/Create、Result+XY、Preset Library 等。

---

## 2. 系统锁定提示词（每一张都先贴）

```text
你正在基于我提供的「拍照页母版」扩展同一套产品 UI。
请严格沿用母版视觉语言，不要重新发明新风格。

硬性规则：
1) 保持一致：圆角体系、边框粗细、按钮形态、底部胶囊导航、控件密度、字体层级。
2) 所有页面都保留同一套底部三键胶囊导航（左/中/右结构一致，中键最突出）。
3) 主流程固定：导入 -> 生成 -> XY 微调 -> 保存/导出。
4) 不允许出现算法下拉、多算法切换、技术术语控件。
5) 不允许出现 Cinematic 开关（仅允许强度滑杆）。
6) 不允许出现 Skin Protection 开关（主界面不展示）。
7) 不使用 emoji 图标，统一使用真实图标风格（线性或实心 SVG 风格）。
8) 只输出一个页面，不要拼多页面长图。

输出必须看起来是同一个产品家族，而不是另一套新设计。
```

---

## 3. 单页生成提示词（按顺序，一张一张做）

### 3.1 拍照页（先校准母版）

```text
基于拍照页母版，生成「Camera Capture」移动端页面（390x844）。

必须包含：
- 顶部状态信息（时间/网络/电量）与轻量操作入口。
- 主取景区域（大圆角，边框与母版一致）。
- 中下部快捷操作工具行（不拥挤）。
- 底部胶囊导航三键：
  左 = 素材/预设入口
  中 = 快门（主按钮）
  右 = 设置

要求：
- 视觉基调简洁，避免 AI 味过重配色（禁蓝紫霓虹）。
- 不使用拟物皮革金属大面积装饰。
- 只输出这一张。
```

### 3.2 首页（Home / Create）

```text
基于同一母版，生成「Home / Create」移动端页面（390x844）。

必须包含：
- 顶部：Kinolu、今日免费额度、升级入口。
- 两个并排主入口按钮（同级，视觉上左右对称）：
  左 = Shoot（直接拍照）
  右 = Edit（进入调色）
- 次入口：Batch Import、My Presets（弱化显示）。
- 文案要让用户理解：这是一个 App 的两个入口，不是两个独立产品。
- 底部胶囊导航沿用拍照页同款组件（尺寸、圆角、按钮比例一致）。

要求：
- 信息层级清晰，保持低决策成本。
- 只输出这一张，不附带别页。
```

### 3.3 结果页（Result + XY）

```text
基于同一母版，生成「Result + XY」移动端页面（390x844）。

必须包含：
- 大预览区，明确 Before/After 切换入口。
- XY 二维控制模块：
  X = Color Strength
  Y = Tone Strength
- XY 可视化要素：
  - 中心死区
  - 可拖拽旋钮
  - 象限提示
  - 实时数值
- 一条 Film Strength 滑杆（无开关）。
- 两个动作：Save Preset、Export。
- 底部胶囊导航同款保留。

限制：
- 不出现算法名称和下拉选择器。
- 只输出这一张。
```

### 3.4 预设库（Preset Library）

```text
基于同一母版，生成「Preset Library」移动端页面（390x844）。

必须包含：
- 预设卡片：缩略图、名称、来源标签（Generated / Imported CUBE）。
- 卡片操作：Apply / Rename / Delete。
- 顶部动作：Import CUBE。
- 分组页签：My Presets / Film Packs。
- 空状态引导文案。
- 底部胶囊导航同款保留。

要求：
- 结构清楚，视觉与拍照页同家族。
- 只输出这一张。
```

### 3.5 批量页（Batch Import）

```text
基于同一母版，生成「Batch Import」移动端页面（390x844）。

必须包含：
- 多文件导入区（拖拽 + 文件选择）。
- 队列列表：缩略图、文件名、状态（waiting/processing/done/failed）。
- 批量控制：选择参考/预设、统一 XY、开始、取消。
- 进度显示：总进度 + 单项进度。
- 结果动作：下载单张、打包下载、重试失败。
- 底部胶囊导航同款保留。

要求：
- 只输出这一张。
```

### 3.6 导出面板（Export Sheet）

```text
基于同一母版，生成「Export Sheet」移动端页面（390x844）。

必须包含：
- 格式：JPG / CUBE / XMP / DNG。
- Pro 锁标识。
- 主 CTA 按钮。
- 小字兼容说明（Lightroom / 视频剪辑工具）。

要求：
- 信息密度高但不乱。
- 只输出这一张。
```

## 4. 防跑偏修正提示词（生成不对就贴）

```text
请严格回到我提供的拍照页母版，不要改动产品风格方向。
本次只修正视觉一致性，不改页面功能结构。

只允许调整：
- 圆角系统
- 边框和阴影强度
- 字重层级
- 胶囊导航细节
- 间距与留白

删除所有新加的风格元素，恢复母版家族一致性。
```

---

## 5. 每张图验收清单（你自己快速检查）

- 是否保持同一套底部胶囊导航（三键结构、比例、中心主按钮）？
- 是否没有算法下拉、没有 Skin Protection 开关、没有 Cinematic 开关？
- 是否是 390x844 单屏，而不是拼图？
- 是否无 emoji，图标风格统一？
- 是否和拍照页在圆角/边框/按钮气质上属于同一家族？

如果任一项不满足，直接用「防跑偏修正提示词」重跑。

## 6. 一次性整套生成（仅单页都过关后再用）

```text
基于我已确认的拍照页母版，生成 Kinolu 全套 6 张移动端原型：
1) Home / Create
2) Result + XY
3) Preset Library
4) Batch Import
5) Export Sheet
6) Editing Panel（Tone/Color/Curves/HSL/Film Finish）

强制要求：
- 所有页面必须共享同一套底部胶囊导航组件样式。
- 视觉语言与拍照页母版一致。
- 不出现算法下拉和技术术语型复杂控件。
- 不使用 emoji 图标。
- 输出附带 token sheet（颜色/字体/间距/圆角/阴影）。
```
